Validator Types Reference

Validator Usage Description
required ['required'] Field must have a value
email ['email'] Field must be valid email format
minLength ['minLength:5'] Field must have minimum N characters
maxLength ['maxLength:100'] Field must have maximum N characters
pattern ['pattern:^[A-Z]'] Field must match regex pattern

Global Configuration

Customize validation messages and defaults globally using the MSV_FORMS_CONFIG injection token.

Usage

Use the provideMsvFormsConfig() helper in your app configuration to override defaults. This is particularly useful for internationalization or custom messaging.

Custom Validators

Beyond string-based validators, use object-based configuration for custom messages and custom validation functions.

Object-Based Validators

Override default error messages per field using object syntax: { type: 'required', message: 'Custom error!' }

Custom Function Validators

Create complex validation logic with custom functions: { type: 'custom', fn: (val) => string | null }
